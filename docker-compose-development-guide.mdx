Develop Zepeed locally with Docker Compose: edit code with hot reload, manage the database, test email, and debug.

## Setup

Complete the [Docker Compose Installation](/deployment/docker-compose-installation), then enable development mode in `.env`:

```env
APP_ENV=local
APP_DEBUG=true
QUEUE_CONNECTION=sync
TELESCOPE_ENABLED=true
```

Restart the app container after changing `.env`, `composer.json`, or `package.json`:

```bash
docker compose restart app
```

## Hot reloading

Start the Vite dev server for instant frontend reloads:

```bash
docker compose exec app npm run dev
```

- Frontend (`.vue`, `.js`, `.css`) — auto-reload
- Backend (`.php`) — manual refresh

Local file edits sync into the container via the bind mount.

## Common tasks

```bash
# Dependencies
docker compose exec app composer install
docker compose exec app npm install

# Lint and format
docker compose exec app composer run analyse
docker compose exec app composer run format

# Tests
docker compose exec app composer run test
docker compose exec app php artisan test --filter testCreateSpeedtest

# Clear all caches
docker compose exec app php artisan optimize:clear
```

## Database

```bash
# Migrations
docker compose exec app php artisan migrate
docker compose exec app php artisan migrate:fresh --seed

# MySQL shell
docker compose exec mysql mysql -u zepeed_user -p zepeed

# Backup and restore
docker compose exec mysql mysqldump -u zepeed_user -p zepeed > backup.sql
docker compose exec mysql mysql -u zepeed_user -p zepeed < backup.sql
```

## Email testing (Mailpit)

Outbound mail is captured by Mailpit. Open the UI at `http://localhost:8025`.

Confirm `.env` points to Mailpit:

```env
MAIL_MAILER=smtp
MAIL_HOST=mailpit
MAIL_PORT=1025
```

## Debugging

**Tinker REPL:**

```bash
docker compose exec app php artisan tinker
```

**Logs:**

```bash
docker compose logs -f app
docker compose exec app tail -f storage/logs/laravel.log
```

**Telescope:** open `http://localhost:8000/telescope` to inspect requests, queries, and exceptions.

## Troubleshooting

**Port 8000 in use** — change the mapping in `compose.yaml`:

```yaml
web:
  ports:
    - "8080:80"
```

**File changes not syncing** — restart the container:

```bash
docker compose restart app
```

**Database connection errors** — check MySQL status:

```bash
docker compose ps mysql
docker compose logs mysql
```

**Asset build fails** — reinstall node modules:

```bash
docker compose exec app rm -rf node_modules
docker compose exec app npm install
```
