Best practices and workflows for developing Zepeed locally using Docker Compose.

## Overview

This guide covers development tasks and workflows when running Zepeed in Docker Compose, including:

- Setting up for local development
- Code editing and hot reloading
- Database management
- Debugging and testing
- Email testing with Mailpit
- Performance optimization

## Development Environment Setup

### Initial Setup

Complete the [Docker Compose Installation](/deployment/docker-compose-installation) guide, then configure for development:

```bash
# Copy development environment
cp .env.dev.example .env

# Edit .env for development
# Set APP_DEBUG=true, APP_ENV=local, etc.
nano .env

# Start services
docker compose up -d

# Initialize database
docker compose exec app php artisan migrate
docker compose exec app php artisan db:seed
```

### Enable Development Features

For faster development iteration, configure these options in `.env`:

```env
APP_ENV=local
APP_DEBUG=true
QUEUE_CONNECTION=sync
CACHE_DRIVER=file
SESSION_DRIVER=file
TELESCOPE_ENABLED=true
```

**Effects**:

- `APP_DEBUG=true` - Show detailed error pages and logs
- `QUEUE_CONNECTION=sync` - Process jobs immediately (no queue worker needed)
- `CACHE_DRIVER=file` - Cache to filesystem (easier to clear)
- `TELESCOPE_ENABLED=true` - Enable Laravel Telescope for debugging

After changes, restart the app container:

```bash
docker compose restart app
```

## Working with Code

### Hot Reloading

Zepeed uses Vite for asset compilation with hot module replacement (HMR).

Start the development server:

```bash
docker compose exec app npm run dev
```

This watches for changes and rebuilds assets automatically:

- **Frontend changes** (`.vue`, `.js`, `.css`) → Instant reload in browser
- **Backend changes** (`.php`) → Requires manual refresh

Keep this terminal open while developing.

### Editing Files

Edit files in your local editor (VS Code, PhpStorm, Vim, etc.). Changes sync automatically to the container via the bind mount.

**File sync works for**:

- PHP files (app, routes, config)
- Vue/JavaScript files
- CSS/Tailwind
- Blade templates

**File sync does NOT include**:

- Dependencies (`composer.json`, `package.json`) - restart container
- Dockerfiles - rebuild container
- Configuration changes in `.env` - restart container

### Common Development Tasks

#### Install PHP dependencies

```bash
docker compose exec app composer install
docker compose exec app composer update some/package
```

#### Install JavaScript dependencies

```bash
docker compose exec app npm install
docker compose exec app npm install -D some-package
```

Or using Bun (faster):

```bash
docker compose exec app bun install
docker compose exec app bun add some-package
```

#### Run code formatters and linters

Check if code follows project standards:

```bash
docker compose exec app composer run analyse    # PHPStan analysis
docker compose exec app composer run format     # Pint + Prettier + ESLint
```

Auto-fix formatting issues:

```bash
docker compose exec app php ./vendor/bin/pint
docker compose exec app npx prettier --write .
```

#### Run tests

```bash
# All tests
docker compose exec app composer run test

# Specific test file
docker compose exec app php artisan test tests/Feature/SpeedtestControllerTest.php

# Single test
docker compose exec app php artisan test --filter testCreateSpeedtest
```

Watch mode for test-driven development:

```bash
docker compose exec app php artisan test --watch
```

#### Generate models and migrations

```bash
# Create a new model with migration
docker compose exec app php artisan make:model Speedtest -m

# Create migration only
docker compose exec app php artisan make:migration create_speedtests_table

# Generate seeders
docker compose exec app php artisan make:seeder SpeedtestSeeder
```

#### Clear application cache

After configuration changes:

```bash
docker compose exec app php artisan cache:clear
docker compose exec app php artisan config:clear
docker compose exec app php artisan view:clear
docker compose exec app php artisan route:clear
```

Or clear all at once:

```bash
docker compose exec app php artisan optimize:clear
```

## Database Development

### Run Migrations

Execute pending migrations:

```bash
docker compose exec app php artisan migrate
```

Rollback last batch:

```bash
docker compose exec app php artisan migrate:rollback
```

Reset database (caution: deletes all data):

```bash
docker compose exec app php artisan migrate:reset
docker compose exec app php artisan migrate:refresh
```

Reset and seed:

```bash
docker compose exec app php artisan migrate:fresh --seed
```

### Seeding

Create sample data using seeders:

```bash
# Run all seeders
docker compose exec app php artisan db:seed

# Run specific seeder
docker compose exec app php artisan db:seed --class=SpeedtestSeeder
```

### Database Inspection

Connect to MySQL with a client:

```bash
docker compose exec mysql mysql -u zepeed_user -p zepeed
```

Or use an external GUI tool:

- **Host**: `localhost`
- **Port**: `3306`
- **Username**: `zepeed_user`
- **Password**: From `.env` file

Query examples:

```sql
-- List tables
SHOW TABLES;

-- View table structure
DESCRIBE speedtests;

-- Count records
SELECT COUNT(*) FROM speedtests;

-- View recent records
SELECT * FROM speedtests ORDER BY created_at DESC LIMIT 10;
```

### Backup and Restore

Backup current database:

```bash
docker compose exec mysql mysqldump -u zepeed_user -p zepeed > speedtests_backup.sql
```

Restore from backup:

```bash
docker compose exec mysql mysql -u zepeed_user -p zepeed < speedtests_backup.sql
```

## Email Testing

### Mailpit Web UI

View emails sent by the application:

```text
http://localhost:8025
```

Access automatically from the web UI when clicking "Send Test Email" in settings.

### Capture Emails

All emails sent by the application go to Mailpit instead of external SMTP servers.

In `.env`, ensure Mailpit is configured:

```env
MAIL_MAILER=smtp
MAIL_HOST=mailpit
MAIL_PORT=1025
MAIL_USERNAME=null
MAIL_PASSWORD=null
MAIL_FROM_ADDRESS="noreply@example.com"
```

### Test Email Sending

Send a test email from the application:

```bash
# Via Laravel Tinker
docker compose exec app php artisan tinker

>>> Mail::raw('Test message', function ($message) {
...   $message->to('test@example.com');
... });
=> true
```

Then check Mailpit UI at `http://localhost:8025` to verify delivery.

### Debugging Email Content

View raw email in Mailpit:

1. Open Mailpit UI: `http://localhost:8025`
2. Click email from list
3. View HTML, text, and headers
4. Copy raw MIME content for debugging

## Debugging

### Laravel Tinker (PHP REPL)

Interactive PHP shell for testing code:

```bash
docker compose exec app php artisan tinker
```

Examples:

```php
// Query database
App\Models\Speedtest::latest()->first();

// Call methods
App\Models\User::find(1)->name;

// Test services
app(App\Services\SpeedtestService::class)->run();

// Check configuration
config('app.debug');

// Exit
exit()
```

### View Logs

Real-time application logs:

```bash
docker compose logs -f app
```

View specific number of lines:

```bash
docker compose logs --tail 100 app
```

View all services:

```bash
docker compose logs -f
```

Laravel logs are written to:

```bash
# View Laravel log directly
docker compose exec app tail -f storage/logs/laravel.log
```

### Enable Query Logging

Log all database queries to file:

In `.env`:

```env
DB_LOG=true
```

Or in code:

```php
// In a controller or service
\DB::listen(function ($query) {
    \Log::info($query->sql, $query->bindings);
});
```

View logs:

```bash
docker compose exec app tail -f storage/logs/laravel.log | grep -i "query"
```

### Browser DevTools

The application runs in the browser, so use standard browser DevTools:

- **Chrome/Edge**: `F12` or `Ctrl+Shift+I`
- **Firefox**: `F12`
- **Safari**: `Cmd+Option+I` (enable in settings first)

Inspect:

- Network requests (speedtest API calls, etc.)
- Console for JavaScript errors
- Application tab for stored data
- Performance for slow operations

### Network Debugging

Inspect HTTP requests between frontend and backend:

```bash
# Capture HTTP traffic
docker compose exec app tcpdump -i eth0 -A 'tcp port 80'

# Or monitor with strace
docker compose exec app strace -e trace=network php artisan serve
```

For easier debugging, use Postman or curl:

```bash
# Test API endpoint
curl -s http://localhost:8000/api/speedtests | jq .
```

## Performance Development

### Monitor Resource Usage

View memory and CPU for running containers:

```bash
docker stats
```

Example output:

```text
CONTAINER       CPU %       MEM USAGE / LIMIT
zepeed_app      0.50%       256MiB / 2GiB
zepeed_mysql    1.20%       512MiB / 2GiB
zepeed_redis    0.10%       25MiB / 1GiB
```

If memory usage is high, increase Docker Desktop limits:

1. Docker Desktop menu → Preferences → Resources
2. Increase Memory slider (default 2GB)
3. Click Apply & Restart

### Profile Code Performance

Use Laravel's built-in profiler (Telescope):

1. Open: `http://localhost:8000/telescope`
2. View requests, queries, exceptions
3. Identify slow operations

Enable in `.env`:

```env
TELESCOPE_ENABLED=true
```

### Cache Debugging

Check what's cached:

```bash
docker compose exec redis redis-cli

redis> KEYS *
redis> GET cache_key_name
redis> TTL cache_key_name
```

Clear Redis entirely:

```bash
docker compose exec redis redis-cli FLUSHALL
```

### Database Query Performance

View slow queries:

```bash
docker compose exec mysql mysql -u zepeed_user -p zepeed

mysql> SELECT * FROM mysql.slow_log\G
```

Enable slow query log in `compose.yaml`:

```yaml
mysql:
  command:
    - --slow_query_log=ON
    - --long_query_time=2
```

### Asset Compilation Performance

Check build times:

```bash
time docker compose exec app npm run build
```

If builds are slow:

1. Check available disk space: `df -h`
2. Increase Docker resources
3. Clear node\_modules: `docker compose exec app rm -rf node_modules && npm install`

## Troubleshooting Development Issues

### Port Conflicts

If port 8000 is already in use:

```bash
# Find what's using port 8000
lsof -i :8000

# Kill the process
kill -9 <PID>
```

Or change port in `compose.yaml`:

```yaml
web:
  ports:
    - "8080:80"
```

### File Sync Not Working

If code changes don't appear in container:

```bash
# Check volume is mounted
docker compose exec app ls -la /var/www/html

# Verify local files exist
ls -la $(pwd)

# Restart container to remount
docker compose restart app
```

### Out of Memory

If containers crash with OOM errors:

```bash
# Check memory usage
docker stats

# Increase Docker Desktop memory
# Docker menu → Preferences → Resources → Memory slider
```

### Database Connection Errors

If the app can't connect to MySQL:

```bash
# Check MySQL is running
docker compose ps mysql

# Check MySQL is healthy
docker compose exec mysql mysqladmin ping -h localhost

# View MySQL logs
docker compose logs mysql

# Restart MySQL
docker compose restart mysql
```

### Asset Compilation Errors

If `npm run dev` fails:

```bash
# Clear node_modules and reinstall
docker compose exec app rm -rf node_modules package-lock.json
docker compose exec app npm install

# Try rebuild
docker compose exec app npm run build
```

### Missing Dependencies

If you see "Class not found" errors:

```bash
# Reinstall Composer packages
docker compose exec app composer install --no-scripts

# Or with full redownload
docker compose exec app composer install --no-cache --no-scripts
```

## IDE Integration

### VS Code Remote - Containers

Develop directly in the container:

1. Install "Dev Containers" extension
2. Open folder in container: `Ctrl+Shift+P` → "Dev Containers: Open Folder in Container"
3. Edit with full IDE features and container environment

### PHP Path Configuration

For IntelliSense and debugging:

In VS Code `settings.json`:

```json
{
  "php.executablePath": "docker compose exec app php",
  "php.validate.executablePath": "docker compose exec app php"
}
```

### PHPStorm / IntelliJ

Configure PHP interpreter:

1. Settings → Languages & Frameworks → PHP
2. Click "..." next to CLI Interpreter
3. Add From Docker Compose
4. Select `compose.yaml` and `app` service
5. Verify PHP version and paths

## Git Workflow

### Exclude Docker Artifacts

`.gitignore` should exclude:

```text
.env
.env.local
.env.*.local
docker/volumes/
storage/logs/
storage/cache/
node_modules/
vendor/
```

### Commit Hooks

Run tests and linters before committing:

```bash
#!/bin/bash
# .git/hooks/pre-commit

docker compose exec -T app composer run test
docker compose exec -T app composer run analyse
```

Make executable:

```bash
chmod +x .git/hooks/pre-commit
```

## Performance Tips

1. **Use Bun instead of npm**: Significantly faster package manager
2. **Enable Docker BuildKit**: `export DOCKER_BUILDKIT=1`
3. **Use named volumes for data**: Better performance than bind mounts on some systems
4. **Limit log verbosity**: Set `APP_DEBUG=false` in production-like tests
5. **Use SQLite for tests**: Faster than MySQL for test databases
6. **Cache dependencies**: Docker layer caching speeds up rebuilds
7. **Minimize container size**: Use Alpine base images where possible
