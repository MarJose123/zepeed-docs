Deploy Zepeed on any Linux host running Docker Engine 24\+ with ports 80 and 443 available.

<Info>
  Tested on **Proxmox LXC (Debian 12)**, **CasaOS**, **Ubuntu Server 24.04**, and **Raspberry Pi OS (64-bit)**.
</Info>

## Prerequisites

| Requirement | Minimum |
| --- | --- |
| Docker Engine | 24\+ |
| Docker Compose | v2.24\+ (plugin, not standalone) |
| RAM | 1 GB |
| Disk | 4 GB |
| DNS | A record for your domain pointing to this host's IP |
| Ports | 80 and 443 open in firewall / NAT |

## Host setup

<Tabs>
  <Tab title="Proxmox LXC">
    <Steps>
      <Step title="Create the container">
        1. In Proxmox, download the **Debian 12** CT template.
        2. Create a new LXC container with:
           - **RAM**: 1024 MB (2048 recommended)
           - **Disk**: 8 GB
           - **Nesting**: enabled (required for Docker)
           - Uncheck **Unprivileged container** — Docker needs a privileged LXC.
        3. Start the container and open its shell.
      </Step>
      <Step title="Install Docker inside the LXC">
        ```bash
        apt update && apt install -y curl
        curl -fsSL https://get.docker.com | sh
        ```
      </Step>
    </Steps>

    Continue with [Common setup](#common-setup) below.
  </Tab>
  <Tab title="CasaOS">
    CasaOS ships with Docker pre-installed. Open the CasaOS terminal (via **Files → Terminal** or SSH) and continue directly with [Common setup](#common-setup).

    <Note>
      CasaOS App Store integration is not yet available for Zepeed. Use the terminal-based setup below.
    </Note>
  </Tab>
  <Tab title="Ubuntu / Debian">
    Install Docker if not already present:

    ```bash
    curl -fsSL https://get.docker.com | sh
    sudo usermod -aG docker $USER   # then log out and back in
    ```

    Continue with [Common setup](#common-setup) below.
  </Tab>
</Tabs>

## Common setup

All steps below are identical regardless of your host OS.

<Steps>
  <Step title="Create a working directory">
    ```bash
    mkdir -p /opt/zepeed && cd /opt/zepeed
    ```
  </Step>
  <Step title="Download the Compose file and env template">
    ```bash
    curl -fsSL https://raw.githubusercontent.com/marjose123/zepeed/main/compose.prod.yml -o compose.prod.yml
    curl -fsSL https://raw.githubusercontent.com/marjose123/zepeed/main/.env.prod.example -o .env
    ```
  </Step>
  <Step title="Configure your environment">
    Open the `.env` file in your editor:

    ```bash
    nano .env
    ```

    **Minimum required values:**

    | Variable | Description |
    | --- | --- |
    | `APP_DOMAIN` | Your domain, e.g. `zepeed.example.com` |
    | `APP_URL` | Full URL, e.g. `https://zepeed.example.com` |
    | `APP_KEY` | Generate using the command below |
    | `ACME_EMAIL` | Email for Let's Encrypt notifications |
    | `DB_ROOT_PASSWORD` | Strong random password for MariaDB root |
    | `DB_PASSWORD` | Strong random password for the app user |
    | `PUSHER_APP_KEY` | Any random string, e.g. `zepeed_key` |
    | `PUSHER_APP_SECRET` | A different strong random string |

    **Generate `APP_KEY`:**

    ```bash
    docker run --rm marjose123/zepeed:latest php artisan key:generate --show
    ```

    Paste the output (starts with `base64:`) into `APP_KEY=`.
  </Step>
  <Step title="Pull and start">
    ```bash
    docker compose -f compose.prod.yml pull
    docker compose -f compose.prod.yml up -d
    ```

    Docker pulls all images, runs database migrations automatically, and starts the application. First boot takes ~60 seconds.
  </Step>
  <Step title="Default admin account">
    A default admin account is created automatically on first boot using the credentials in `.env`:

    | Variable | Default |
    | --- | --- |
    | `DEFAULT_ADMIN_EMAIL` | `admin@zepeed.local` |
    | `DEFAULT_ADMIN_PASSWORD` | `zepeed_admin` |

    <Warning>
      Change `DEFAULT_ADMIN_PASSWORD` in `.env` **before** starting the stack. After login, update your credentials under **Profile Settings**.
    </Warning>

    On every subsequent restart, the account creation step is skipped automatically.
  </Step>
  <Step title="Open in browser">
    Navigate to `https://your-domain.com`. Let's Encrypt issues a certificate automatically on first request (takes ~10 seconds).
  </Step>
</Steps>

## Updating Zepeed

```bash
cd /opt/zepeed
docker compose -f compose.prod.yml pull
docker compose -f compose.prod.yml up -d
```

Migrations run automatically on startup — no manual steps needed.

<Tip>
  To pin a specific version, set `ZEPEED_VERSION=1.1.0` in `.env` before pulling.
</Tip>

## Useful commands

```bash
# View live logs
docker compose -f compose.prod.yml logs -f app

# Open an artisan shell
docker exec -it zepeed-app php artisan tinker

# Check container health
docker compose -f compose.prod.yml ps

# Stop everything (data is preserved in volumes)
docker compose -f compose.prod.yml down

# Full teardown including all data (irreversible)
docker compose -f compose.prod.yml down -v
```

## Backup

All persistent data lives in named Docker volumes:

| Volume | Contents |
| --- | --- |
| `zepeed_zepeed-mariadb` | Database |
| `zepeed_zepeed-storage` | Uploaded files |
| `zepeed_zepeed-logs` | Application logs |
| `zepeed_zepeed-letsencrypt` | TLS certificates |

**Back up the database:**

```bash
docker exec zepeed-mariadb \
  mariadb-dump -u root -p"${DB_ROOT_PASSWORD}" zepeed \
  > zepeed-backup-$(date +%Y%m%d).sql
```

**Restore:**

```bash
docker exec -i zepeed-mariadb \
  mariadb -u root -p"${DB_ROOT_PASSWORD}" zepeed \
  < zepeed-backup-YYYYMMDD.sql
```

## Troubleshooting

### App won't start or health check failing

```bash
docker compose -f compose.prod.yml logs app --tail=50
```

### Database connection refused

Check that `DB_PASSWORD` in `.env` matches `DB_ROOT_PASSWORD` and that `mariadb` is healthy:

```bash
docker compose -f compose.prod.yml ps mariadb
```

### Let's Encrypt certificate not issuing

- Confirm port 443 is reachable from the internet (`curl https://your-domain.com`).
- Confirm the DNS A record resolves to this host's IP.
- Check Traefik logs:
  ```bash
  docker compose -f compose.prod.yml logs traefik
  ```

### WebSocket not connecting

Confirm `PUSHER_APP_KEY` and `PUSHER_APP_SECRET` in `.env` match `SOKETI_DEFAULT_APP_KEY` and `SOKETI_DEFAULT_APP_SECRET`. They are set from the same variables, so they should always match unless you edited only one.
