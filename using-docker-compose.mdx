title: "Using Docker Compose"

Install and run Zepeed using Docker Compose for a consistent, isolated environment across development, staging, and production deployments.

## Overview

Docker Compose enables you to define Zepeed and its dependencies (PHP, Node.js, MySQL, Redis) in a single `compose.yaml` file, eliminating environment inconsistencies and simplifying deployment.

This guide covers:

- Complete Docker Compose setup
- Environment configuration
- Database initialization
- Application startup
- Production deployment considerations
- Troubleshooting

## Setup

<Steps>
  <Step title="Clone repository">
    Clone the Zepeed repository to your local machine.

    ```bash
    git clone https://github.com/MarJose123/zepeed.git
    cd zepeed
    ```
  </Step>
  <Step title="Create environment configuration">
    Copy the example environment file for your target environment. For development:

    ```bash
    cp .env.dev.example .env
    ```

    For production:

    ```bash
    cp .env.prod.example .env
    ```

    The environment file contains configuration for the application, database, cache, mail, and external services.
  </Step>
  <Step title="Configure environment variables">
    Edit the `.env` file to match your deployment requirements. At minimum, set:

    ```env
    APP_NAME="Zepeed"
    APP_KEY=base64:YOUR_APP_KEY
    APP_ENV=production
    APP_DEBUG=false
    
    DB_CONNECTION=mysql
    DB_HOST=mysql
    DB_PORT=3306
    DB_DATABASE=zepeed
    DB_USERNAME=zepeed_user
    DB_PASSWORD=secure_password
    
    CACHE_DRIVER=redis
    REDIS_HOST=redis
    REDIS_PORT=6379
    ```

    <Warning>
      In production, never commit `.env` to version control. Use secrets management tools (Kubernetes Secrets, Docker Secrets, HashiCorp Vault) instead.
    </Warning>
  </Step>
  <Step title="Start containers">
    Build and start all services in detached mode.

    ```bash
    docker compose up -d
    ```

    Docker Compose will:

    1. Build the Zepeed application image
    2. Pull MySQL, Redis, and Mailpit images
    3. Create containers and networks
    4. Start all services
  </Step>
  <Step title="Initialize database">
    Run database migrations and seed initial data.

    ```bash
    docker compose exec app php artisan migrate
    docker compose exec app php artisan db:seed
    ```

    <Note>
      If migrations fail with "No such table" errors, ensure the MySQL container is healthy before retrying. Check with `docker compose ps`.
    </Note>
  </Step>
  <Step title="Generate application key">
    If you haven't set `APP_KEY` in your `.env` file, generate it now.

    ```bash
    docker compose exec app php artisan key:generate
    ```

    Update your `.env` file with the generated key and restart the application:

    ```bash
    docker compose restart app
    ```
  </Step>
  <Step title="Verify installation">
    Open your browser and navigate to the application URL:

    ```text
    http://localhost
    ```

    You should see the Zepeed login page. Log in with default credentials (if seeded) or create a new account via the application interface.

    Verify services are running:

    ```bash
    docker compose ps
    ```

    Expected output shows all containers with status "Up".
  </Step>
</Steps>

## Services

The default Docker Compose stack includes these services:

### App (Laravel Application)

Runs the Zepeed application using PHP-FPM.

**Image**: Built from project `Dockerfile`\
**Port**: 9000 (internal)\
**Features**:

- Laravel 13 with Inertia.js
- Vue 3 frontend
- Automatic asset compilation with Vite
- Database migrations on startup

### Web (Nginx)

Reverse proxy serving HTTP requests to the application.

**Image**: `nginx:latest`\
**Port**: 80 (mapped to 8000 locally)\
**Configuration**: `docker/nginx.conf`

### MySQL

Relational database storing application data.

**Image**: `mysql:8.0`\
**Port**: 3306\
**Volume**: `mysql_data` persists across restarts\
**Default Database**: `zepeed`

<Warning>
  MySQL 8.0 requires explicit authentication plugin configuration. The default authentication plugin `mysql_native_password` is enabled for compatibility.
</Warning>

### Redis

In-memory cache and session store.

**Image**: `redis:alpine`\
**Port**: 6379\
**Volume**: `redis_data` persists across restarts

### Mailpit

Local SMTP server and web UI for email testing.

**Image**: `axllent/mailpit:latest`\
**SMTP Port**: 1025\
**Web UI**: `http://localhost:8025`

Use Mailpit to preview emails sent by the application without exposing external SMTP credentials during development.

### Port Mapping

Default port mappings (from container to host):

| Service | Container Port | Host Port | Purpose |
| --- | --- | --- | --- |
| Web (Nginx) | 80 | 8000 | Application UI |
| MySQL | 3306 | 3306 | Database access |
| Redis | 6379 | 6379 | Cache/session access |
| Mailpit | 1025, 8025 | 1025, 8025 | SMTP & email preview |

To modify ports, edit the `ports` section in `compose.yaml` or use environment variable overrides:

```bash
docker compose -f compose.yaml -f compose.override.yaml up -d
```

Create `compose.override.yaml`:

```yaml
services:
  web:
    ports:
      - "8080:80"
  mysql:
    ports:
      - "3307:3306"
```

### Volumes

Persistent storage for application data and services:

| Volume | Mount Path | Purpose |
| --- | --- | --- |
| `mysql_data` | `/var/lib/mysql` | Database persistence |
| `redis_data` | `/data` | Cache persistence |
| `.` | `/var/www/html` | Application source code |

Volumes are stored in Docker's data directory. Inspect with:

```bash
docker volume ls
docker volume inspect zepeed_mysql_data
```

### Custom Nginx Configuration

Edit `docker/nginx.conf` to customize server behavior:

```nginx
# Example: Add custom headers
add_header X-Frame-Options "SAMEORIGIN" always;
add_header X-Content-Type-Options "nosniff" always;
 
# Example: Configure caching
location ~* \.(jpg|jpeg|png|gif|ico|css|js|woff)$ {
    expires 30d;
}
```

After changes, rebuild the web service:

```bash
docker compose up -d --build web
```

## Common Tasks

### View logs

Display logs from a specific service or all services.

```bash
# All services
docker compose logs -f
 
# Specific service
docker compose logs -f app
 
# Last 50 lines
docker compose logs --tail 50 app
```

### Execute artisan commands

Run any Laravel artisan command from the app container.

```bash
docker compose exec app php artisan tinker
docker compose exec app php artisan queue:work
docker compose exec app php artisan schedule:run
```

### Access database

Connect to MySQL from the host machine:

```bash
docker compose exec mysql mysql -u zepeed_user -p zepeed
```

Or use an external client (TablePlus, DBeaver, MySQL Workbench):

- **Host**: `localhost` or `127.0.0.1`
- **Port**: `3306`
- **Username**: `zepeed_user`
- **Password**: Check your `.env` file

### Rebuild application image

Rebuild the app container when the Dockerfile or dependencies change.

```bash
docker compose up -d --build app
```

### Clear application cache

```bash
docker compose exec app php artisan cache:clear
docker compose exec app php artisan config:clear
docker compose exec app php artisan view:clear
```

### Backup database

Export the MySQL database to a SQL file:

```bash
docker compose exec mysql mysqldump -u zepeed_user -p zepeed > zepeed_backup.sql
```

When prompted, enter your database password from `.env`.

### Restore database

Import a previously exported SQL file:

```bash
docker compose exec mysql mysql -u zepeed_user -p zepeed < zepeed_backup.sql
```

### Stop containers

Stop all running containers without removing them:

```bash
docker compose stop
```

Resume with:

```bash
docker compose start
```

### Remove containers and volumes

Completely remove all containers, networks, and volumes:

```bash
docker compose down -v
```

<Warning>
  This permanently deletes all data in MySQL and Redis. Always backup before running this command in production.
</Warning>

## Production Deployment

### Security Hardening

Before deploying to production, implement these security measures:

<Steps>
  <Step title="Use a reverse proxy">
    Deploy Zepeed behind a reverse proxy (Nginx, Apache, Caddy) with:

    - SSL/TLS encryption
    - DDoS protection
    - Rate limiting
  </Step>
  <Step title="Disable debug mode">
    Set `APP_DEBUG=false` in `.env` to prevent sensitive information leaks.
  </Step>
  <Step title="Manage secrets">
    Never store secrets in `.env` files in version control. Use:

    - Kubernetes Secrets (Kubernetes deployments)
    - Docker Secrets (Docker Swarm)
    - Environment variable injection at runtime
    - Secrets management services (HashiCorp Vault, AWS Secrets Manager)
  </Step>
  <Step title="Enable database authentication">
    Use strong passwords for MySQL users and restrict network access.

    ```sql
    ALTER USER 'zepeed_user'@'%' IDENTIFIED BY 'strong_password_here';
    FLUSH PRIVILEGES;
    ```
  </Step>
  <Step title="Configure persistent storage">
    Replace named volumes with bind mounts pointing to persistent storage:

    ```yaml
    services:
      mysql:
        volumes:
          - /mnt/data/mysql:/var/lib/mysql
    ```
  </Step>
  <Step title="Set resource limits">
    Prevent resource exhaustion by adding memory and CPU limits to services:

    ```yaml
    services:
      app:
        deploy:
          resources:
            limits:
              cpus: '2'
              memory: 1G
            reservations:
              cpus: '1'
              memory: 512M
    ```
  </Step>
</Steps>

### Scaling

For high-traffic deployments, use Docker Swarm or Kubernetes to orchestrate multiple Zepeed instances behind a load balancer. The current single-instance Docker Compose setup is suitable for small to medium deployments.

### Backup Strategy

Automate daily backups of the MySQL database:

```bash
#!/bin/bash
# backup.sh
 
BACKUP_DIR="/path/to/backups"
DATE=$(date +%Y%m%d_%H%M%S)
 
docker compose exec -T mysql mysqldump \
  -u zepeed_user \
  -p"${DB_PASSWORD}" \
  zepeed > "${BACKUP_DIR}/zepeed_${DATE}.sql"
```

Schedule with cron:

```bash
0 2 * * * /path/to/backup.sh
```

## Troubleshooting

### Container fails to start

Check logs for error messages:

```bash
docker compose logs app
docker compose logs mysql
```

Common issues:

- **Port already in use**: Change port mapping in `compose.yaml`
- **Insufficient disk space**: Clean up with `docker system prune`
- **Permission denied**: Ensure Docker daemon is running and you have permissions

### Database connection fails

Verify MySQL is running and healthy:

```bash
docker compose ps mysql
docker compose exec mysql mysql -u root -p -e "SELECT 1"
```

If MySQL fails to start, check data directory permissions:

```bash
docker compose exec mysql chmod 755 /var/lib/mysql
```

### Application errors after startup

Wait for MySQL to finish initialization. The first startup takes longer:

```bash
docker compose logs mysql | grep "Ready for connections"
```

Then verify the app is running:

```bash
docker compose restart app
docker compose logs app
```

### Slow performance

Monitor resource usage:

```bash
docker stats
```

If containers are resource-constrained, increase Docker's allocated memory and CPU in Docker Desktop settings.

### Changes to code not reflected

For development, enable file watching:

```bash
docker compose exec app npm run dev
```

For production assets:

```bash
docker compose exec app npm run build
docker compose restart app
```

### Reset to clean state

Remove all containers, volumes, and networks, then rebuild:

```bash
docker compose down -v
rm -rf docker/volumes
docker compose up -d
docker compose exec app php artisan migrate
docker compose exec app php artisan db:seed
```
